# Figma Make Light Theme Follow-Up Prompt

```text
Stay in the same Figma session and use the dark theme portfolio design you already completed as the source of truth.

Now create the full light theme version of that same portfolio system.

Important:
- This is not a new redesign from scratch.
- This is not a simple color inversion.
- Preserve the same core concept, design intelligence, information architecture, component logic, and overall brand feeling.
- The light theme should feel like the same product family as the dark theme, with the same level of polish, intentionality, and originality.

Working mode:
- Do not ask me to choose between multiple light-theme directions unless absolutely necessary.
- Use your best judgment and commit to the strongest light-theme execution for the concept you already established.
- Keep the layout, structure, content hierarchy, and overall creative direction consistent unless a small adjustment is genuinely necessary to make the light theme better.
- If some dark-theme-specific effects do not translate well, reinterpret them intelligently instead of forcing them unchanged.

What I need:
- A complete light theme across the entire portfolio system you already designed
- The same pages, sections, components, states, and responsive views that exist in the dark theme
- A light theme that feels native, premium, and fully resolved, not secondary or compromised

Light theme quality bar:
- Maintain strong contrast and readability without becoming flat or generic
- Avoid drifting into bland white SaaS visuals
- Preserve the distinct personality of the concept
- Rework backgrounds, surfaces, accents, dividers, shadows, glows, overlays, and depth cues so they are truly designed for light mode
- Rebalance typography color, emphasis, and hierarchy where needed
- Ensure CTA prominence still works in light mode
- Make form fields, cards, navigation, and interactive elements feel intentional in light mode

Required scope:
- Main portfolio experience
- All major sections already created in the dark theme
- Desktop version
- Mobile version
- 404 / not found page
- Contact area and its important states
- Navigation states
- Reusable component patterns
- Key interaction states including hover, focus, active, and success/error treatments where relevant

Non-negotiable completion rule:
- Do not stop after updating only the homepage or hero
- Do not leave secondary pages, mobile views, or component states unfinished
- Do not end the session until the light theme is fully applied across the same scope as the dark theme
- If the dark theme includes a complete system, the light theme must also be a complete system

Theme system requirements:
- Make dark and light feel like two equally intentional modes of the same brand
- If a theme toggle exists, both modes should feel first-class
- If token changes are useful, define the light-mode equivalents for the major visual decisions
- Keep accessibility and usability strong in both modes

What to preserve:
- The overall concept and visual DNA
- The narrative structure and content hierarchy
- The premium, technically credible tone
- The distinctiveness of the design

What you may improve:
- Any details that need adjustment so the light theme works better
- Contrast, spacing, border treatment, elevation, or accent behavior if the dark-theme choices do not translate directly
- Any small design decisions required to make the system feel equally strong in light mode

Final instruction:
Use the completed dark theme as your base, then fully extend that same system into a finished light theme without leaving gaps.

Final self-check before ending:
- Is the light theme complete across the same full scope as the dark theme?
- Are desktop and mobile both covered?
- Are all key pages and states covered, including 404 and contact states?
- Do the two themes feel like one coherent high-quality product family?
- If any answer is no, continue working instead of ending the session.
```
